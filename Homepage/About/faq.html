FAQ

Here are frequently asked questions, mainly on mailing list and irc

How can I contact you?

See here.

I there a tutorial

Yes, there is a tutorial for new programmers, check it out on our Wiki.

Status of elektra?

The library is usable now for people which are willing to follow the development of elektra. You fill find rarely bugs here and there. It is not recommended for stable use until we release the version 1.0.

Is there a long term roadmap?

  • Before end of year 2006: Cleanup API, Improve Testframework, finish Berkley and Ini Backends
  • Before end of year 2007: Release 1.0 for at least 5 platforms (Linux, BSD, Windows, 2 other Unixes) Very well tested on each of them, ready for public use.

What are backends?

Backends mean that the library can write different configurations formats. The configuration can have on one system ini style format and on another XFree86 for all apps.

I am an Enduser, what is this all about?

All configurations of apps using elektra can be browsed graphical like in kde and gnome now, but independ of the desktop environment.

I am an Admin, what is this all about?

You can browse the configuration with a command-line tool kdb or simple edit it by hand. It even will be possible to edit it over http like webmin or with a graphical user interface.

What language does libelektra uses?

libelektra uses C, also for the backends. But apps using elektra can be written in any of the language bindingssupported languages.

I mean natural languages!

It is not planned that the homepage, documentation or source code has any other language then english. But the documentation of the keys, the guis and apps will certainly support i18n. The enduser will have anything in their language. People developing elektra simple have to learn english.

I want this or that feature

Simple implement it and send us a patch. If it is good work we will integrate it 100%. But don't annoy us with dreams or whatever. We are working on a stable, well tested API and backends now and not on new features.

Once we have reached 1.0 we are willing to implement more new stuff in a new devel branch created then. Thenwe can talk about ACL support and a new notification API.
