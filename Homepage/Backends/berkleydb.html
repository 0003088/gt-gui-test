Berkleydb

Developer: Avi Alkalay

Status

This backend lacks some features.

About

The whole System-Database is stored into a single file. Every User has also its own Database in his Homedirectory.

Thus the right management in the filesystem is managed by berkleydb, it will be on a minimum level of security. It is not possible that a user can change any system-values if you want a minimum of security.

With berkleydb you get the fastest and smallest system possible. It uses the highly optimized database routines from berkleydb.

Aim

This Backend can be used on single user machines or where users don't have to change any system variables and one administrator(root) is fully responsible. It is also perfectly suited for embedded systems, systems without network, or systems where no users are logged in.

Editing

It is not possible to edit the database by hand. Use kdb for it.
