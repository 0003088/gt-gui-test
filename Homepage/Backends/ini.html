ini

Developer: Markus Raab

This is a ini-style backend. Key/Value Pairs are stored in files in following scheme: key1=value1;comment

The design goal is to have a situation as today, but unified. There should be per application one configuration file in $HOME and in /etc.

From security standpoint it is between filesys (most secure possible) and single configuration file (like berkleydb).

Ini makes it possible that users can edit their own config files.
